# 03 UI

## 목적

검색 사용자 화면과 색인 상태 대시보드의 사용자 흐름·화면 책임·접근성 기준을 정의한다.

## 적용 기준

- 클라이언트가 검색을 호출하는 간단한 검색 화면과, 색인 작업 상태를 확인하는
  대시보드를 제공한다. 화면은 별도 배포 단위 `web-ui`로 둔다
  ([ADR-0001](../deliverables/adr/0001-architecture-and-module-boundary.md)).
- `web-ui`는 api-service의 REST API만 소비하고 백엔드 로직을 중복 구현하지 않는다.
- 검색 화면: 질의 입력, 결과·점수·소요시간 표시. 대시보드: 작업 상태·재시도·실패
  표시(FR-6, 09 운영과 연결).
- 오류·로딩·빈 결과 상태와 기본 접근성(키보드·대비)을 처리한다.
- 프론트엔드 기술 선택은 후속 단계에서 확정한다(이력서 전략상 후순위).

## 완료 기준

- 검색 화면과 색인 대시보드의 책임·API 경계가 정의되어 있다.
- 화면이 api-service API만 소비한다.
- 오류·접근성 처리 기준이 있다.
