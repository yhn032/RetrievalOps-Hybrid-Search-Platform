# 평가 계약 (M0·M2·M4)

검색 품질과 서버 성능을 동일 조건에서 전후 비교하기 위한 측정 계약이다. 모든
결과 수치에는 테스트 환경·데이터 크기·반복 횟수를 함께 명시하며, 목표 미달
결과도 병목과 다음 개선안과 함께 공개한다. 요구사항은
[요구사항 문서](../derived/side-project-requirements.md)를 따른다.

## M0 — 범위와 평가 계약

| 항목 | 계약 |
|---|---|
| 데이터셋 | 공개 데이터셋만 사용. 라이선스·재배포 가능 여부를 확인하고 출처를 기록한다. 최종 선정은 M0 실행 시 ADR로 확정한다. |
| 질의·정답셋 | 평가용 질의와 정답(관련 문서) 집합을 정의한다. |
| 평가셋 분리 | 색인·튜닝에 쓰는 집합과 평가 집합을 분리해 데이터 누수를 방지한다. |
| 측정 환경 | 하드웨어·데이터 크기·동시성·모델 버전을 고정하고 기록한다. |
| 데이터 스키마 | 문서·작업 메타데이터 스키마와 API 계약 초안을 정의한다. |

## M2 — RAG Retrieval 평가

동일 질의셋에서 4개 구성을 비교한다: BM25, Dense, Hybrid(BM25+Dense), Hybrid+Reranker.

| 지표 | 정의 | 비고 |
|---|---|---|
| Recall@k | 상위 k개 안에 정답이 포함된 비율 | k ∈ {1, 5, 10} |
| MRR | 첫 정답의 역순위 평균 | |
| nDCG@k | 순위 가중 누적 이득의 정규화 값 | k ∈ {10} |

결과표 필수 기록 항목: 데이터 크기, 임베딩·Reranker 모델, 청크 설정(크기·중복),
구성별 지표 값. 평가셋은 M0에서 분리한 평가 집합만 사용한다.

완료 기준:

- BM25·Dense·Hybrid·Reranker 결과를 같은 질의셋·같은 표로 비교했다.
- 평가셋 분리로 누수를 방지했다.
- 결과표에 데이터 크기·모델·청크 설정이 기록되어 있다.

## M4 — 성능·관측·배포 측정

부하 테스트는 k6 또는 동등 도구를 사용한다.

| 지표 | 정의 |
|---|---|
| 지연시간 p50/p95 | 검색 요청 응답시간 분포 |
| QPS | 초당 처리 질의 수 |
| 오류율 | 실패 응답 비율 |
| 색인 처리량 | 단위 시간당 색인 문서 수 |
| 색인 실패율 | 색인 작업 실패 비율 |
| 캐시 적중률 | 캐시로 처리된 응답 비율 |

측정 조건:

- 동일 하드웨어·데이터·동시성에서 개선 전후를 비교한다.
- 워밍업 수행 여부와 반복 횟수를 기록한다.
- p50/p95·오류율·색인 처리량을 함께 공개한다.
- 목표 미달 결과도 숨기지 않고 병목과 다음 개선안을 기록한다.

## 공통 원칙

- 측정하지 않은 값(예: 실제 운영 QPS·사용자 수)은 결과로 제시하지 않는다.
- 모든 수치는 테스트 환경·데이터 크기·반복 횟수를 동반한다.
- 목표값(예: 목표 Recall@k, p95 임계)은 M0 실행 시 기준선 측정 후 ADR로 확정한다.
